建立自己的私有云存储

发表于 2015-10-31   |   分类于 服务器   |  

操作系统：ubuntu 14.04
软件平台：owncloud

1. 预装环境软件

ownCloud 内核是用PHP5写的，支持SQLite、MySQL、Oracle以及PostgreSQL等数据库。本教程使用MySQL开源数据库。系统需要安装的软件如下：

  • web 服务器：apache2
  • php 安装包：php5, php5-gd, php-xml- parser,php5-intl
  • 数据库驱动：php5-mysql
  • curl 安装包：curl, libcurl3, php5-curl
  • smb 客户端：smbclient （挂载windows共享文件夹的）
  • 安装MySQL：mysql-server

3. 环境配置

第1步：修改owncloud目录权限
ownCloud需要对一些目录有写的权限，为此，web服务器用户必须要拥有apps、data、config目录的权限。

第2步：配置web服务器
我们要为owncloud配置Apache服务器，owncloud需要启用Apache上.htaccess文件，.htaccess文件(或者”分布式配置文件”）提供了针对目录改变配置的方法。为了启用web服务器上的.htaccess，可以通过 Apache配置文件的 AllowOverride指令来设置。

mod_rewrite模块提供了实时地将访问者请求的静态URL地址映射为动态查询字符串的一种规则。

4. 运行

在浏览器中打开 http://localhost/owncloud ，你会被引导进入ownCloud的配置界面。
第1步：创建管理员用户。
在界面上输入用户名和密码。
第二步：连接数据库。
选择mysql数据库，输入MySQL的数据库名、用户名和密码。（数据库owncloud 用户root 密码为安装数据库时输入的）

5. 静态ip添加（非必须）

不设置静态IP无法通过静态IP访问，也即通过http://192.168.1.200/owncloud（假设我的IP为192.168.1.200）无法访问owncloud。可通过修改owncloud配置文件解决。

6. 使用桌面同步客户端同步文件

owncloud提供windows、mac 、linux、手机端客户端软件，安装很简单，在此就不再啰嗦。连接服务器地址填写 http://192.168.1.200/owncloud
注：客户端下载地址

7. 公网访问（其实木有必要，私有云嘛）

上述搭建的owncloud服务器是基于局域网的，换句话说，就是在局域网才能访问。要想通过外网访问，必须做路由器端口映射或者挂上自己云主机。端口映射方式成本比较低，可用花生壳软件做映射，不过近几年来花生壳越来越坑了，不给钱估计就只剩下壳了（不稳定，几分钟掉一次线）。如果知道自己公网IP(可通过网络查询，但不一定能用，ping测试一下)，可直接做路由器映射，但是必须具有路由器管理权限；云主机方式成本较高，需要有云主机（最低配置也得几十刀一个月，流量也就10来g，流的都是血啊）。
端口映射方式（路由器）：

1. 获取IP（http://www.ip138.com/），ping 测试一下，不通就木有办法了。或者打10000调戏妹子问去（PS：据说可以，但是木有试过）。
2. 登录路由器（http://192.168.1.1/）
3. 在转发规则下的虚拟服务器下添加映射条目（填写好端口号和公网IP）
4. 重启一下路由就ok 了

云主机方式：
把上面的步骤在自己的云主机ECS上重新做一遍，只有在第4步添加IP不一样而已，把公网ip加入owncloud的配置文件（/var/www/owncloud/config/config.php）就ok了，不过得注意防火墙拦路虎（Ubuntu默认关闭防火墙，或者压根就木有）。
